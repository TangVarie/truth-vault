## 1. 架构总览

```
飞书投放表(多项目)                      Railway worker(LLM)
   │  daily-sync.yml(夜间 cron 02:00 UTC)      ▲ essence/curate
   ▼                                            │
truth_vault.*(Supabase)  ──essence标注/子方向──┘
   │  notes / metric_snapshots / projects / comments / flywheel_*
   ├─ 通道1:爆款 → sanshengliubu.reference_samples(给 vibe 仿写)  ✅在跑
   ├─ 飞轮:爆款 → 解构成"经验卡"(flywheel_lesson_annotations)→ LLM 馆员 write-time 借阅  ✅
   └─ public.v_dash_*(18 张只读聚合视图)──anon──► Next.js 看板(Vercel)
```

- **数据来源**:飞书多维表格,每项目一张表;字段经 `mappings/<project>.yaml` 映射进 `truth_vault.notes`。
- **结构化(essence)**:LLM 按受控词表(`05`)给每条打标(情绪杠杆/人性原型/受众/内容形态/时效…),跑在 Railway worker 上(GitHub 海外 runner 连不上中转站,D-038)。
- **两个通道**:
  - **通道 1 = ssll(sanshengliubu)**:TV 验证级爆款 → `reference_samples`,喂 vibe 仿写。**在跑**。
  - **通道 2 = autowriter push**:**已退役(D-038,2026-06-01)**。改为 **pull / 馆员模型**:写稿时 LLM 馆员从"经验卡库"按需借阅最相关的卡。见 `14`。
- **前端**:三层,详见 §5。
