# 05 · 受控词表 v0.1

## 为什么存在

Essence 层和 Audience 层的标注必须基于**闭集词表**。LLM 自由表达跨样本不可比，跨项目无法对比。这个文档定义所有受控词表的 v0.1 版本。

**词表版本**: v0.1 (草案，待 Ziao + 周哥 review)  
**最后更新**: 2026-05-18

> 词表是 Truth Vault 的核心约束。词表错了 essence 层就废。所以 v0.1 必须从策略视角过一遍。

---

## 词表演化规则

1. **版本号永远递增**（v0.1 → v0.2 → v1.0）
2. **每条数据记录标注时的词表版本**（`essence_vocab_version` 字段）
3. **添加新值容易，删除现有值困难** —— 删除需要先把所有使用此值的历史数据迁移
4. **每次修改词表必须创建新决策记录**（DECISIONS.md）

## 1. `intent` · 内容意图（5 个值）

最稳定的分类 —— 来自飞书表已有的「发布笔记」字段。

| 值 | 中文 | 定义 | 飞书原值 |
|---|---|---|---|
| `traffic` | 流量向 | 内容主导，产品仅作为载体或不出现，目的是获取流量 | 流量帖 / 钓鱼帖 |
| `conversion` | 产品向 | 产品是主角，直接推荐购买、强调功效 | 直给笔记 / 小红书下单 / 淘宝下单 |
| `educational` | 教育向 | 科普类，介绍知识或方法 | 科普贴 |
| `mixed` | 混合 | 同时承担流量和转化（少见） | (推断) |
| `other` | 其他 | 不属于以上类型 | 截图 / 大字报 |

**review 重点**: 这个划分够吗？是否需要"互动向"（评论区互动）等独立类别？

## 2. `content_format` · 内容形式（8 个值）

文案的结构层组织方式。

| 值 | 定义 | 典型特征 |
|---|---|---|
| `情感叙事` | 以故事/场景/情节为主体，有人物、对话、情感转折 | 第一人称、场景描写、情感独白 |
| `认知重构` | 反认知共识，提出新观点，重塑读者认知 | "其实..."、"原来..."、"很多人不知道..." |
| `横评对比` | 多个产品/方案的对比评测 | 表格、列表、A vs B、红黑榜 |
| `教程攻略` | 教方法、给步骤的实用内容 | "第一步...第二步..."、"教你"、"必看" |
| `直给推荐` | 直接推产品，强调好处 | "推荐"、"必入"、"好用炸了" |
| `场景植入` | 在特定使用场景中自然带出产品 | "出差时"、"加班后"、"约会前" |
| `提问求助` | 以提问/求助开头，引导评论 | "求姐妹推荐"、"有没有人..." |
| `反差破圈` | 通过强反差/意外感吸引点击 | "没想到"、"竟然"、"反转" |

**review 重点**: 
- 是否覆盖所有项目？
- "反差破圈"和"情感叙事"的边界够清楚吗？

---

## 3. `emotional_lever` · 情绪杠杆（10 个值）⭐ 最重要

文案触发反应的根本情绪机制。这是 essence 层最核心的字段。

### 负向 4 件套（撬动负面情绪）

| 值 | 定义 | 典型 angle |
|---|---|---|
| `焦虑撬动` | 激发对未来/现状的不安 | "再不...就晚了"、"35+ 了还..." |
| `羞耻撬动` | 激发自我形象受损感 | "被嫌弃"、"丢人"、"不好意思" |
| `恐惧撬动` | 激发对具体威胁的恐惧 | 健康威胁、社交威胁、形象威胁 |
| `愤怒撬动` | 激发对外部对象的不满 | 对行业、对群体、对现象 |

### 正向 4 件套（建立正面情绪）

| 值 | 定义 | 典型 angle |
|---|---|---|
| `造梦投射` | 描绘理想生活/未来 | "成为...的样子"、"过上...的生活" |
| `认同感建立` | "你不是一个人"的共鸣 | "懂的都懂"、"姐妹们都这样" |
| `归属感建立` | 建立群体身份认同 | "我们 35+ 女性"、"宝妈群体" |
| `共鸣释放` | 表达被压抑的情绪 | "终于敢说"、"憋了好久" |

### 中性 2 件套

| 值 | 定义 | 典型 angle |
|---|---|---|
| `好奇驱动` | 激发探索欲 | "竟然"、"原来"、"没想到" |
| `信息差利用` | 提供独家/稀缺信息 | "内部消息"、"行业内幕"、"少有人知" |

**review 重点（关键讨论点）**:
- 10 个值是否覆盖了帆谷历史项目的所有 angle？
- "焦虑撬动"和"恐惧撬动"边界是否够清楚？建议判断标准：焦虑是模糊的未来不安，恐惧是具体的威胁
- 是否需要新增："虚荣撬动"（intentionally let user feel superior）？

---

## 4. `emotional_valence` · 情绪极性（3 个值）

| 值 | 定义 |
|---|---|
| `positive` | 正向（造梦、认同、归属、共鸣释放、好奇） |
| `negative` | 负向（焦虑、羞耻、恐惧、愤怒） |
| `neutral` | 中性（信息差） |

通常和 `emotional_lever` 强相关，自动推断。但允许 LLM 显式标注（有的内容情绪复杂）。

---

## 5. `emotional_intensity` · 情绪强度（3 个值）

| 值 | 判断标准 |
|---|---|
| `low` | 情绪杠杆存在但较弱，读起来不刺激 |
| `medium` | 中等强度，能让读者停下来 |
| `high` | 强烈情绪触发，有"破防"感 |

参考标尺：
- 力克雷"相亲被嫌弃一身烟味后我彻底醒了" → high
- 力克雷"NRT有用吗？" → low
- HXZ_FB "55 岁了，求好用又实惠的粉饼" → medium

---

## 6. `human_truth_archetype` · 人性原型（17 个值）⭐

文案触动的深层人性原型。**支持 1-2 个值（主 + 次）**。这是穿越周期的核心字段。

### 关系类

| 值 | 定义 |
|---|---|
| `同辈比较` | 和同龄人/同阶层的对比压力 |
| `伴侣关系` | 恋爱、夫妻、亲密关系 |
| `代际冲突` | 与父母/子女/长辈的差异 |
| `职场关系` | 同事、领导、下属关系 |

### 自我类

| 值 | 定义 |
|---|---|
| `自我形象维护` | 对外形象的焦虑/管理 |
| `身份认同` | 我是谁、我属于哪个群体 |
| `时间流逝感` | 对年龄/衰老/时间的感受 |
| `自由意志` | 摆脱束缚、不被定义 |

### 焦虑类

| 值 | 定义 |
|---|---|
| `阶层焦虑` | 财富、地位、社会等级 |
| `经济焦虑` | 收入、消费、生活成本 |
| `健康焦虑` | 身体、疾病、衰老 |
| `育儿焦虑` | 孩子成长、教育、未来 |

### 缺失类

| 值 | 定义 |
|---|---|
| `情感缺位` | 缺少陪伴、被理解 |
| `归属缺失` | 不属于任何群体 |
| `认同缺失` | 不被看见、不被认可 |

### 欲望类

| 值 | 定义 |
|---|---|
| `控制感渴望` | 想掌控自己的生活 |
| `自我提升` | 想变得更好、aspiration 驱动 |

**review 重点**:
- 17 个是否够？是否需要"宠物相关"（NUC 有相关角度）、"消费愉悦"？
- 边界：`身份认同` vs `归属缺失` 的差异是什么？

## 7. `trend_dependencies` · 时效依赖（多选，9 个值）

标识这条文案依赖哪些时效性元素。**用于判断时间衰减速度**。

| 值 | 定义 |
|---|---|
| `特定平台事件` | 引用具体小红书/抖音热点事件 |
| `特定IP引用` | 提到具体明星、综艺、电影、电视剧、网红 |
| `时事热点` | 引用当前新闻、社会事件 |
| `季节性事件` | 双 11、春节、夏季等 |
| `当代流行语` | 用当下流行词（"绝绝子"、"米色裤子"等） |
| `节日` | 节假日相关 |
| `行业事件` | 品类内特定事件（如美妆周、戒烟日） |
| `平台话术` | 强依赖某平台特定语言（小红书"姐妹们"等） |
| `通用` | 不依赖任何时效性元素，可穿越周期 |

**用法**: 一篇文案可以同时含多个标签。`通用` 是排他的（含通用则不含其他）。

**review 重点**:
- 为什么 `通用` 单独存在？因为大多数文案都依赖某些时效，"通用"是稀有但极其重要的样本（最易迁移）。

## 8. `target_audience` · 目标受众（10 个值）

**项目级 + 笔记级双层** —— 项目 onboarding 时定义项目级，每条笔记 essence 标注时可细化。**支持多选**。

| 值 | 定义 | 典型场景 |
|---|---|---|
| `年轻女性` | 18-29 女性 | 独居、新职场、消费升级 |
| `中年女性` | 30-49 女性 | 职场中坚、二宝妈、年龄焦虑 |
| `银发女性` | 50+ 女性 | 健康、子女、跳广场舞 |
| `年轻男性` | 18-29 男性 | 独居、健身、初入社会 |
| `中年男性` | 30-49 男性 | 职场压力、健康焦虑 |
| `银发男性` | 50+ 男性 | 慢病管理、子女 |
| `学生党` | 在校生（大学为主） | 性价比、宿舍、考试 |
| `宝妈` | 育儿期女性 | 婴幼儿、产后恢复 |
| `伴侣家人` | 不是产品使用者，是为别人买的人 | 戒烟药、保健品给长辈 |
| `通用` | 不区分目标人群 | 横评、科普类 |

**review 重点（关键 Q1）**: 是否需要把"宝妈"细分为"育儿早期"vs"育儿中期"vs"二胎宝妈"？

**review 重点**: 城市层级（一线/新一线/三四线）要不要进 audience 字段？建议放 `inferred_audience_profile` JSONB 里的 demographic 部分，不进闭集。

## 9. `inferred_audience_profile` · 受众画像（JSONB 结构）

不是闭集字段，是结构化 JSONB。结构如下：

```yaml
inferred_audience_profile:
  # ── 闭集部分（统计可比）──
  demographic:
    age_band: ["30-39", "40-49"]      # 多选，受控值 [20-29, 30-39, 40-49, 50+]
    gender_skew: female               # female / male / mixed
    city_tier: ["1线", "新1线"]        # 多选，受控值 [1线, 新1线, 2线, 3-4线, 5线及以下]
    life_stage: "育儿期"               # 闭集: 学生 / 职场新人 / 已婚未育 / 育儿期 / 空巢 / 退休
    value_orientation: "精致"          # 闭集: 务实 / 精致 / 自洽 / 表达 / 反叛
    income_band: "中产"                # 闭集: 学生 / 入门 / 中产 / 高净值
  
  # ── 自由文本部分（信息密度优先）──
  psychographic:
    primary_pain: "孩子教育焦虑 + 自我职场焦虑"
    primary_aspiration: "希望成为有品味、不被定义的中年女性"
    likely_objections: "价格、是否真的有效"
  
  # ── 元数据 ──
  confidence: 0.7                      # LLM 推断置信度（自评 0-1）
  inferred_at: "2026-05-18T12:00:00"
```

闭集部分用于 SQL 统计（"50+ 女性人群在哪些品类有偏好"），自由文本部分用于人类阅读和深度分析。

## 10. `tier` · 内容表现等级（7 个值）

笔记最终表现的人工标注 label。

| 值 | 定义 | 来源 |
|---|---|---|
| `趴` | 没有水花，互动很少 | 状态字段"无水花" / 备注"淘汰" / 数值低 |
| `预备` | 有爆款潜质但还在观察 | 状态字段"爆贴预备" |
| `爆` | 已经爆了，互动显著超过项目均值 | 状态字段"爆贴" / 备注"新爆" / 数值高 |
| `大爆` | 超级爆款，互动是中位数的 50-100 倍 | 状态字段"大爆" |
| `风控` | 被平台限流或风控 | 状态字段"风控中" |
| `删除` | 已被删除（人工或平台） | 备注"删0" / "他自己删了" |
| `未知` | 还在观察 / 数据不足 | 状态字段"评估中" / 完全无数据 |

**训练时使用建议**:
- 二分类（爆 vs 不爆）：`爆 + 大爆` → 正样本；`趴` → 负样本；`预备 / 风控 / 删除 / 未知` → 不用
- 多分类：`趴 / 爆 / 大爆`三分类，其他状态不用

## Review 工作流

Ziao + 周哥 review 这份词表的建议工作流：

### Step 1（30 分钟）· 单独 review

每人独立看完文档，标出：
- 不理解的值
- 觉得需要新增/合并/删除的值
- 边界模糊的地方

### Step 2（45 分钟）· 一起讨论

按词表顺序过一遍：
- 1. intent → 通常通过
- 2-3. content_format / emotional_lever → 重点讨论
- 4-5. valence / intensity → 通常通过
- 6. human_truth_archetype → 重点讨论
- 7. trend_dependencies → 通常通过
- 8-9. target_audience / inferred_audience_profile → 讨论是否细分

### Step 3（15 分钟）· 输出 v0.2

把 review 的修改整理进新版本：
- 添加值：直接加
- 修改边界：更新定义
- 删除值：检查是否有历史数据使用，决定迁移方案
